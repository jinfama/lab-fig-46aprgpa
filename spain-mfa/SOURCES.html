Convenciones de columnas

  • Path: ruta relativa al raíz del proyecto.
  • Bucket: subcarpeta de data/raw/ (origen lógico).
  • Type: rol funcional (input | reference | legacy-fallback).
  • Status: ver tabla anterior.
  • Size (bytes): tamaño en disco (output de _compute_source_checksums.py).
  • SHA-256: hash para detectar cambios silenciosos.
  • Notes: cobertura, regenerador, citas, advertencias.

3. Citas externas (provenance de fuentes primarias)

mfa/ no extrae primarias propias: consume CSVs producidos por mining/, agriculture/, forest/ y trade/ (snapshots regenerables en data/processed/upstream/). La cadena 3-tier completa (archivo → paper → primarias) vive en cada uno de esos datasets upstream (p. ej. forest/docs/FUENTES.md, mining/docs/SOURCES.md). Las primarias subyacentes principales son:

  • Estadística Minera de España (1860–2017) — vía mining/. Alimenta MF2/MF3/MF4.
  • Anuarios de Estadística Agraria (MAPA, 1929–presente) — vía agriculture/. Alimenta MF11x cultivos, MF12x residuos/forraje.
  • Anuario de Estadística Forestal MITECO + IFN1–IFN4 — vía forest/. Alimenta MF131/MF132.
  • Estadísticas comerciales españolas + Eurostat Comext — vía trade/. Alimenta IMP/EXP.
  • Eurostat env_ac_mfa — referencia de validación independiente, en references/.
  • MFA legacy v2.0 (legacy_seriemfa/) — versión anterior CAHE, fallback temporal y referencia de validación interna.

4. Mantenimiento

  1. Tras añadir/reemplazar un archivo en data/raw/, ejecutar: bash python docs/_compute_source_checksums.py # imprime tabla python docs/_compute_source_checksums.py --write # actualiza docs/_sources_checksums.md Y reflejar la fila nueva o el nuevo SHA-256 en este documento.
  2. Renderizar el HTML navegable: bash python docs/_render_sources_html.py
  3. Para los snapshots upstream en data/processed/upstream/, no se cataloga aquí: la provenance se mantiene en el dataset upstream que los produce y la trazabilidad de regeneración está en code/load_from_upstream/ + docs/methodology/00_load_from_upstream.md.
