1. Qué es WHEP

Base de datos histórica mundial que integra fuentes del siglo XIX al XXI en un panel armonizado país × año × indicador para cultivos (área, producción, rendimientos) y comercio (bilateral, directo). Pensada para alimentar un Scientific Data data descriptor y depósito Zenodo. Autor: Juan Infante-Amate (Universidad de Granada).

7. Cómo usarlo

Cargar un panel

import sys; sys.path.insert(0, "D:/data/whep")
from code import setup as S
import pandas as pd

df = pd.read_csv(S.FINAL / "crops" / "whep_crops_balanced_panel.csv")

Regenerar el pipeline desde cero

cd D:/data/whep
python code/parsing/parse_mitchell_extras.py   # ejemplo
python code/integration/integrate_crops.py
python code/panels/build_level2_panel.py
python code/validation/validate_series.py
python code/publishing/publish_final.py

Compilar el paper

cd papers/whep_data_descriptor/_build
pdflatex paper.tex && pdflatex paper.tex
mv paper.pdf ../manuscript.pdf

8. Cómo añadir cosas nuevas

Una fuente nueva

  1. Carpeta data/raw/{bucket}/{nombre_fuente}/ con archivo(s) original(es) exactos como vinieron.
  2. Entrada en docs/SOURCES.md con autor, URL, año, cita, checksum SHA256 (vía docs/_compute_source_checksums.py), provenance_status.
  3. code/parsing/parse_{nombre_fuente}.py que produce data/processed/cleaned_by_source/{nombre_fuente}_clean.csv.
  4. code/integration/integrate_*.py que la incorpora al panel.
  5. Entrada en docs/methodology/DECISIONS.md si la incorporación implica decisiones metodológicas no triviales.
  6. Línea en CHANGELOG.md.

Un indicador nuevo

  1. Decidir si requiere un módulo de pipeline propio o reutiliza el existente.
  2. Crear figures/{indicador}/_code/ con scripts numerados estilo /fig-nature.
  3. Añadir columna en los paneles finales correspondientes.
  4. Documentar en docs/architecture.md (lista de indicadores) y en README.md.

9. Decisiones del usuario vigentes

  • Compartimentación por indicador, no por tema. No invertir.
  • Lenguaje: Python (excepción al default R del PI). Mantener.
  • Estilo de figuras: /fig-nature siempre. Sin excepciones.
  • data/raw/ intocable. Las correcciones viven en code/cleaning/.
  • Cero alucinación en fuentes. Bloquear publicación de todo lo marcado unverified.
  • Versiones viejas a _archive/, nunca borrar.
